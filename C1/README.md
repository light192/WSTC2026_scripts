# C1 PNETLab Checker

Авточекер для оценки задания C1 в PNETLab. Основной файл запуска:
`c1_check_ios.py`.

## Подготовка

1. Запустите нужную лабораторию в PNETLab.
2. Имя running-сессии должно содержать строку `module-c`.
3. Все Cisco-устройства, которые проверяются, должны быть включены.
4. В каталоге проекта должны быть файлы:
   - `creds.json` - доступ к PNETLab и учетные данные устройств;
   - `topo.json` - ожидаемая адресация и интерфейсы реальной топологии.

## creds.json

Файл должен содержать такие поля:

```json
{
  "pnet_url": "http://192.168.1.21",
  "username": "checker",
  "password": "checker",
  "device_username": "admin",
  "device_password": "password",
  "enable_password": "password"
}
```

`username` и `password` используются для входа в PNETLab.
`device_username`, `device_password` и `enable_password` используются для входа
на Cisco-устройства.

## Базовый запуск

Из каталога проекта:

```powershell
python .\c1_check_ios.py
```

По умолчанию проверка начинается с критерия `A` и после каждого аспекта ждет
нажатия `Enter`.

## Запуск без пауз

```powershell
python .\c1_check_ios.py --no-pause
```

## Запуск с нужного места

Можно начать проверку с критерия, подкритерия или номера аспекта.

```powershell
python .\c1_check_ios.py --start C
python .\c1_check_ios.py --start C2
python .\c1_check_ios.py --start 23
```

Допустимые значения:

- критерии: `A`, `B`, `C`, `D`, `E`, `F`;
- подкритерии: `A1`, `A2`, `B1`, `B2`, `B3`, `C1`, `C2`, `C3`,
  `D1`, `D2`, `D3`, `E1`, `E2`, `E3`, `F1`, `F2`, `F3`, `F4`, `F5`, `F6`;
- номера аспектов: `1`-`57`.

Пример запуска с подкритерия и без пауз:

```powershell
python .\c1_check_ios.py --start F3 --no-pause
```

## Что делает чекер

1. Логинится в PNETLab.
2. Находит running-сессию лаборатории по строке `module-c`.
3. Присоединяется к сессии.
4. Открывает постоянные консоли ко всем Cisco-устройствам в начале проверки.
5. Перед проверками готовит консоль: `terminal length 0`, `terminal width 511`,
   `line console 0`, `exec-timeout 0 0`.
6. Выполняет команды проверки с фильтрами и читает вывод до возврата IOS prompt.
7. После каждого устройства сразу выводит:
   - команду проверки;
   - вывод с устройства;
   - результат `PASS` или `FAIL`.
8. После аспекта показывает общий результат по устройствам и начисленные баллы.

Для BGP scorer не сравнивает строки `show running-config`. Соседства, AS,
таймеры и `next-hop-self` проверяются по operational-командам `show bgp` /
`show ip bgp`, чтобы корректно засчитывать peer-groups и другие эквивалентные
варианты конфигурации.

## Статусы

- `PASS` - аспект выполнен полностью, начисляется полный балл.
- `PART` - аспект выполнен частично, балл пересчитывается пропорционально.
- `FAIL` - аспект не выполнен, балл не начисляется.
- `SKIP` - аспект нельзя честно проверить только через IOS-консоли.

Для частичного результата, например `5/7`, балл считается так:

```text
балл аспекта * 5 / 7
```

Частичный результат подсвечивается сиреневым и отображается в формате:

```text
[PART] Row 20 (C) 0.54/0.75 (5/7)
```

## Итоговый отчет

В конце выводится:

- результат каждого аспекта;
- сумма по критериям `A`-`F`;
- общий `TOTAL`.

Цвета итогов:

- зеленый - полный балл;
- сиреневый - частичный балл;
- желтый - есть `SKIP`;
- красный - балл не набран;
- голубой - критерий не запускался, например при старте с середины.
