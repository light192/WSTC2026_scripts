# C2 — эталонное ручное решение

Этот каталог содержит эталонную конфигурацию Training Task C2 для сравнения с
решением участника. Конфигурации рассчитаны на опубликованную PNETLab-топологию.

## Важное расхождение в исходных материалах

В таблице задания ошибочно указаны IR2 G0/4 (WAN) и G0/5 (IR3). На фактической
схеме и в лаборатории используются:

- IR2 G0/3 — ISP1, `203.0.113.6/30`;
- IR2 G0/4 — IR3, `10.80.0.41/30`.

Именно эти интерфейсы использованы в эталоне. Перед применением можно подтвердить
их командой `show cdp neighbors` и по физической схеме PNETLab.

## Состав пакета

- `configs/` — отдельный paste-ready файл для каждого настраиваемого Cisco;
- `hosts/PC1-PC4.txt` — получение DHCP и приемочные тесты VPCS;
- `hosts/SVR1-SVR2_ISP1.txt` — состояние преднастроенных узлов, без их изменения;
- `PORT_MAP.md` — назначение физических портов;
- `VALIDATION.md` — команды сравнения и ожидаемые результаты;
- `apply_order.txt` — безопасный порядок ручной настройки.

## Как применять

1. Не изменять ISP1, SVR1 и SVR2: по условию они уже настроены.
2. Открыть консоль нужного устройства, перейти в `enable`.
3. Вставить содержимое его файла из `configs/`.
4. Проверить ошибки IOS. Команды Flexible/Classic NetFlow могут различаться у
   разных образов; в файлах используется Classic NetFlow для IOSv.
5. Выполнить `copy running-config startup-config`.
6. После настройки всех устройств выполнить `VALIDATION.md`, затем C2 checker.

Пароли эталона: пользователь `admin`, enable/login secret `Skill39@C2`;
SNMPv3 auth `C2Auth@2026`, privacy `C2Priv@2026`.

На IR3 применяется static TCP/80 mapping `198.51.100.89:80` →
`10.81.130.10:80`. Это точно соответствует функциональной проверке `/healthz`
и, в отличие от безусловного one-to-one mapping, не переводит межсайтовый
private traffic OPS-SRV, для которого явно требуется NAT exemption.

## Границы эталона

Файлы задают требуемое функциональное состояние. Служебные строки, которые IOS
создает автоматически (`version`, `boot-start-marker`, сертификаты и т.п.),
намеренно не включены. Sticky MAC появятся после трафика клиентов и потому не
фиксируются заранее.
