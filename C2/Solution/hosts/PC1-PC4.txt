C2 VPCS — ручная настройка и приемочные тесты
================================================

Статические адреса запрещены. На каждой VPCS-консоли выполнить:

  ip dhcp
  show ip

Ожидается:
  PC1  10.80.10.100-199/24  GW 10.80.10.1  DNS 10.81.130.10
  PC2  10.80.40.100-199/24  GW 10.80.40.1  DNS 10.81.130.10
  PC3  10.82.10.100-199/24  GW 10.82.10.1  DNS 10.81.130.10
  PC4  10.83.10.100-199/24  GW 10.83.10.1  DNS 10.81.130.10

DNS (в VPCS имя разрешается командой ping):

  ping ops.c2.skill39.local -c 2
  ping judge.c2.skill39.local -c 2
  ping internet-test.c2.skill39.local -c 2

Ожидаемые адреса: 10.81.130.10, 10.81.150.10, 198.51.100.100.

Разрешенная матрица:

  PC1 -> PC3, PC4, 10.81.130.10, 198.51.100.100, 8.8.8.8
  PC3 -> PC1, 10.81.130.10, 198.51.100.100, 8.8.8.8
  PC4 -> PC1, 10.81.130.10, 198.51.100.100, 8.8.8.8
  PC2 -> 10.81.130.10, 198.51.100.100, 8.8.8.8

Запрещенная матрица PC2 (ожидается timeout):

  PC1, PC3, PC4, 10.81.150.10, 10.80.50.21, 10.82.50.21, 10.83.50.21

После DHCP renew проверить bindings на AS1-AS4:

  show ip dhcp snooping binding

Для генерации NAT/NetFlow выполнить с каждого PC ping 198.51.100.100 и 8.8.8.8.
