# C3 PNETLab Checker

Автоматизированная оценка Training C3 по 100 Measurement aspects (25 баллов).
Основной файл — `c3_check_ios.py`, критерии — `c3_criteria.tsv`.

## Запуск

Заполните `creds.json`, запустите лабораторию PNETLab и выполните:

```powershell
cd D:\Worldskills\WSTC2026\WSTC2026_scripts\C3
python .\c3_check_ios.py
```

После подключения скрипт выводит все активные сессии PNETLab. Введите ID
нужной лаборатории из показанного списка. Поиск по части имени лаборатории
не используется.

Для запуска без интерактивного выбора можно заранее указать ID:

```powershell
python .\c3_check_ios.py --session-id 3
```

Запуск без пауз или с определенного аспекта:

```powershell
python .\c3_check_ios.py --no-pause
python .\c3_check_ios.py --start D13
python .\c3_check_ios.py --start G
```

Недеструктивный режим, в котором F2/F4/F7/F8/F9 помечаются `SKIP`:

```powershell
python .\c3_check_ios.py --skip-disruptive
```

## Принципы проверки

- Используются operational и специализированные команды `show`.
- `show running-config` не используется.
- Единственное разрешенное исключение — `show startup-config | include ^ip route`
  в A7 для обнаружения неактивных static routes, которые нельзя надежно увидеть
  в RIB. Это прямо предусмотрено официальной схемой оценки.
- Для каждого аспекта выводятся точные ожидаемые свойства, команды, фактический
  ответ устройства и результат по каждому устройству.
- Частично выполненные составные аспекты получают пропорциональный балл.
- Управляемые отказные тесты обязательно восстанавливают интерфейс или BGP
  neighbor в блоке `finally`.

## Источники требований

`c3_criteria.tsv` сформирован из
`C3_Marking_Scheme_WSC2026_RU_Revised.xlsx` и содержит все 100 аспектов:
описание, проверяемые свойства, максимальный балл и официальный How to Mark.

Скрипт использует проверенные библиотеки подключения C1/C2. Файл `host_lib.py`
оставлен в каталоге C3 для дальнейшего автономного развития host-проверок.
